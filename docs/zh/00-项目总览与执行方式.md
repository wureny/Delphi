# Delphi 项目总览与执行方式（v0.1）

## 1. 项目背景
Delphi 是一个基于本体论（Ontology）的多智能体协同系统，服务于 Agentic Fund 的研究、决策与执行流程。

当前阶段优先级：
1. 先完成 Ontology（数据层之上的语义与关系层）。
2. 再建设多 Agent 协同系统。

## 2. 当前阶段目标（Phase 1）
聚焦对 Polymarket 进行 Ontology 化，形成：
1. 统一的实体定义与关系定义。
2. 可被程序消费的数据结构与约束规则。
3. 可追踪的开发任务与里程碑。

## 3. 执行原则
1. 产品先行：先把 PRD、范围、验收标准写清楚。
2. 分层推进：Ontology 层先于 Agent 协同层。
3. 小步迭代：每个 Issue 尽量 1-3 天可完成并可验证。
4. 文件系统即项目记忆：文档、决策、进展都落在仓库内。

## 5. 项目管理节奏（建议）
1. 每周一：更新里程碑和本周 issue 清单。
2. 每天：在 issue 中更新进展和 blocker。
3. 每周末：更新 `plans/milestones` 下的阶段复盘。
