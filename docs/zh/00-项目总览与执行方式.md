# Delphi 项目总览与执行方式（v0.2）

## 1. 项目背景
Delphi 是一个基于本体论（Ontology）的多智能体协同系统，服务于 Agentic Fund 的研究、决策与执行流程。

当前阶段优先级（截至 2026-02-28）：
1. 在已落地 ontology + multi-agent runtime skeleton + paper trading v0 基础上，补齐执行审计链路与安全评测。
2. 完成从“可运行”到“可管理、可评估”的收口。

## 2. 当前阶段目标（Phase 2）
聚焦“执行域闭环与治理收口”，形成：
1. `DecisionRecord -> RiskPolicy -> Order -> Execution -> Position/PnL` 的 paper trading 闭环。
2. 可审计的执行链路（decision/evidence/gate/execution 的可追溯引用）。
3. 与实现同步的里程碑、issue 状态和周节奏项目管理资产。

## 3. 执行原则
1. 产品先行：先把 PRD、范围、验收标准写清楚。
2. 分层推进：Ontology 层先于 Agent 协同层。
3. 小步迭代：每个 Issue 尽量 1-3 天可完成并可验证。
4. 文件系统即项目记忆：文档、决策、进展都落在仓库内。

## 5. 项目管理节奏（建议）
1. 每周一：更新里程碑和本周 issue 清单。
2. 每天：在 issue 中更新进展和 blocker。
3. 每周末：更新 `plans/milestones` 下的阶段复盘。

## 6. 下一步（按优先级）
1. 推进 G7：补齐 execution audit replay 与一致性校验，固化审计链路质量门禁。
2. 推进 G8：建立 recommendation quality + execution safety benchmark，并纳入回归流程。
3. 回补 G6：增强 simulation 长运行稳定性与报表维度（在 v0 闭环上加固）。
4. 每周更新一次阶段状态到 `plans/milestones/`（建议新增周状态文件）。
