# Thread Workspaces

这里给 `dev_plan.md` 里的每个 thread 提供一个独立文档工作区。

目标不是在这里写完整内容，而是提前把后续多线程协作需要的骨架准备好。

每个 thread 目录统一包含：

- `brief.md`
  - 这个 thread 要解决什么
- `deliverables.md`
  - 这个 thread 最终要产出什么文档或结果
- `notes.md`
  - 过程中的草稿、问题、候选方案
- `handoff.md`
  - 交接给其他 agent 时要说明什么

使用规则：

- thread 只修改自己负责的工作区和它明确拥有的 canonical docs
- 如果 thread 想改项目边界，先回到 foundation / deep dive 文档处理
- `notes.md` 可以乱一点，`brief.md` 和 `deliverables.md` 要尽量稳定
